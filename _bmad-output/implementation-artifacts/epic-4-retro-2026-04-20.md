## Challenges

1. **ATDD inline-stub pattern produced tests that didn't exercise real code.** 4.2 and 4.3 both shipped initial ATDD files where `buildTestApp()` inlined the route logic rather than importing `src/routes/*.js`. Tests "passed" without loading the production plugin. Caught in review both times and rewired (`120a49d` for 4.2, `e9840f2` for 4.3). This is a generation-phase ATDD pattern — the pre-emptive test design hasn't yet learned to import from the real module path. Forward-only fix: update BAD ATDD prompt to require real-plugin imports.

2. **PR body hallucination persists across three epics.** PR #46 body overstated `Content-Type` (claimed `text/csv`; actual is `text/csv; charset=utf-8`) and claimed the route "enforces" the 12-column CSV header contract (it doesn't). Same BAD Step 6 pattern flagged in Epic 2 and Epic 3 retros. Bounded by `bad-review` diff-auditing; no BAD-level fix required beyond a prompt tweak.

3. **Rate-limiting deferred three times in one epic.** 4.1 (POST /api/generate), 4.2 (polling — real DoS amplification with ~1–2s cadence per tab), and 4.3 (CSV TEXT-column read). Each review deferred with identical "belongs in platform hardening story" reasoning. Queue now has four cross-cutting rate-limit findings.

4. **CSV formula injection (CWE-1236) is the only live defect in the deferred queue.** `escapeCell()` in `src/workers/scoring/buildReport.js:31-43` applies RFC 4180 quoting but does not neutralise `=`/`+`/`-`/`@`/`\t`/`\r` prefixes on text cells. `product_title` is attacker-controllable via any competitor listing on Worten. When Gabriel opens the report CSV in Excel, `=HYPERLINK(...)` or DDE-style payloads execute. Fix is low-cost but was deferred because a blanket neutraliser would break exact-byte ATDD fixtures. Defers to Story 3.5 retrofit.

5. **Sprint-status `epic-4` row still `in-progress` at retro start — same root cause, second occurrence.** Phase 0 reconciliation works at story-level; epic-level rollup still isn't automatic. This is the second retro in a row where we flip the epic row by hand.

6. **Defense-in-depth queue is growing.** Response JSON schema on `/api/jobs/:id`, oversized-`job_id` route-level guard, `csv_data = null` handling, `Cache-Control: private, no-store`, CSV UTF-8 BOM assertion, control-character log scrubbing test. None are live defects — all belt-and-suspenders against future regressions. Accepted for carry but worth naming.

## Key Insights

1. **Retro action items that compound are worth more than retros that catalogue.** Every Epic 3 commitment shipped before Epic 4 dev began, and the real-key integration test justified itself in its very first run. The pattern — retro surfaces a gap, gap becomes gate, gate catches the next real bug — is the compounding loop.

2. **External-API epics need real-credential gates, not retro afterthoughts.** The Mirakl auth header bug would have surfaced in production had the integration test not run pre-Epic-4. Promote to hard gate for any epic that touches external APIs (Mirakl, Resend).

3. **Thin layers over solid layers stay thin.** Epic 4's entire value proposition is "HTTP routes over a worker". Zero business logic in routes. Zero transformations. Minimal schema validation. The review findings were architectural (ordering, rollback) or additive (hardening queue) — never "re-do the thing". The Epic 3 retro's "boring reliability compounds" insight holds: a clean Epic 3 made a quiet Epic 4 possible.

4. **ATDD generation has a real-import blindspot.** Inline-stub routes in `buildTestApp()` passed tests without exercising production code — twice. The pattern is mechanical (pre-emptive test file has no real source to import yet when generated) but forward-only fixable via prompt update.

5. **`bad-review` is now the meta-safety-net.** Three iterations during one epic, catching PR body hallucinations that `dev-story` produced. The review loop is what makes the decorative-PR-body stance viable. Don't replace `bad-review` by rewriting BAD; tune the `dev-story` prompt and let `bad-review` keep catching the rest.

6. **Defects and defense-in-depth belong in different queues.** CSV formula injection is a live CWE-classified bug. Rate limiting, Cache-Control, response schemas, BOM tests — all speculative hardening. Conflating them in the same deferred-work list dilutes urgency. The retro-driven action items distinguish the two explicitly.
