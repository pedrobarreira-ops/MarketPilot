## Challenges

1. **Intra-epic test-granularity inconsistency — no convention for Epic 6 to inherit.** Story 5.1 consolidated 17 planned scenario IDs into 4 bundled Playwright tests. Story 5.2 went the opposite direction — 7 tests, roughly one-per-scenario for 19 ACs. Both pass, neither is wrong. Epic 6 has 6 stories × 5-10 ACs each; without a convention, each dev agent will re-decide independently. **Resolved in this retro — see Action Item P1.**

2. **Static ATDD scope creep on 5.1 — architecture invariants embedded in story-level ATDD.** Story 5.1's static file added 3 architecture invariants beyond story ACs (no server-side imports, no eval/document.write, no innerHTML user injection). Defensible as general-purpose frontend guards, but borderline against the `feedback_atdd_static_scans.md` rule (runtime invariants for ordering claims; structural only for security/architecture). If Epic 6's 6 stories each embed their own invariant set, story ATDDs bloat 6× without cross-referencing. **Resolved in this retro — see Action Item P2.**

3. **Tailwind Play CDN JIT is a cross-cutting Epic 6 landmine.** PR #52 surfaced it: `progressFill.classList.add('bg-red-600')` didn't paint red because Tailwind Play CDN JIT purges class names that don't appear as static text in the HTML. Fix was inline `style.backgroundColor = '#DC2626'` alongside the class. Epic 6 is *all* data-driven UI — first-row tints on opportunities tables, red `#DC2626` gap cells, WOW score horizontal bars, PT/ES toggle aria-pressed styling, colour-coded cells. Without a project-wide rule, every 6.x story will re-discover this. **Resolved in this retro — see Action Item T1.**

4. **PR body imprecise phrasing slipped past the Step 6 hallucination guard.** PR #51 body stated the static ATDD "covers 10 acceptance criteria" when it actually covers 2 ACs via 10 scan assertions. The guard catches function names, Portuguese strings, file paths, navigation URLs — but *count-category labelling* (tests vs. assertions vs. ACs) is a newer, subtler failure mode. Not a correctness issue, but the "PR body as decorative, diff as source of truth" stance relies on catching these. **Resolved in this retro — see Action Item P3.**

5. **E2E coverage gaps deferred on Story 5.2 — four distinct patterns.** (a) AC-12 fallback link (blocked-navigation branch) never simulated; (b) count boundary cases (`progress_current=0`, `progress_total=0`, `current > total`) not tested at status-line render layer; (c) polling network-error resilience (5xx, timeout, malformed JSON mid-poll) not test-enforced despite dev notes claiming "continue polling"; (d) copy-button `execCommand` fallback (browsers without clipboard permission) never exercised. All deferred to post-Epic-6 hardening. None live defects — all carry-forward risk.

6. **`.additional.test.js` mixed static + behavioural on 3.5a.** Pre-Epic-5 but flagged for this retro per `feedback_atdd_static_scans.md`. 3.5a's file has 19 behavioural tests plus 5 keyword-grep structural-classification tests. The story test plan allowed them; the memory rule is stricter. Resolution candidates: tighten memory to allow "structural classification" explicitly, or leave as case-by-case judgment. **See discussion under Action Item P2** — resolved by the shared-invariants-file move (removes the pressure entirely).

## Key Insights

1. **Retro action items that compound are worth more than retros that catalogue — second proof point.** Every Epic 4 commitment shipped, and every commitment generated a concrete Epic 5 save: Phase 0 auto-flip eliminated manual sprint-status editing (`9abad27`); Playwright scaffold eliminated mid-story infra wiring; Step 6 prompt caught the big PR-body hallucinations. The compounding loop — retro surfaces gap → gap becomes gate → gate catches the next real issue — is now validated across three consecutive epics.

2. **Pre-epic scaffold investment is a standing rule, not a one-off.** Playwright harness in `84e2e96` was the quietest ~30-minute investment of the sprint. Epic 6 inherits it for free. The pattern generalises: any epic that introduces a new surface area (DOM tests, new API type, new storage layer) should ship its test scaffolds in the *previous* epic's prep work, not mid-story. See Action Item P4.

3. **Meta-workflow skills benefit from in-flight iteration, not speculative pre-design.** `bad-review` v5 shipped three iterations during Epic 5's PR stream — mechanical conflict resolution (PR #51 save), Phase 1 cross-reference step (PR #52 self-validated), Phase 3 dedup against prior deferred items. Each iteration landed *because* a real PR surfaced a real gap. Pre-designing these in one shot would have missed the patterns that only emerged under live workflow pressure. The PR cadence is the test suite for process skills.

4. **Test-enforcement converts invariants from convention to contract.** Epic 5's static ATDD scans — `no localStorage`/`sessionStorage`, URL-assignment-before-`setInterval`, no `eval`/`document.write` — mean the invariants fail fast if anyone refactors them away. This is the same compounding pattern as Epic 3's pino redact test-enforcement. Project-wide invariants (Tailwind dynamic class, no cross-seller JOINs, `api_key` never in HTTP response) should all graduate to test-enforced status as they're discovered. Action Item T1 puts the first frontend-architecture invariants file in place; more will follow.

5. **Solo-dev debug economics change which trade-offs win.** Per-scenario Playwright tests cost more lines but save more time on failure investigation when the dev is also the reviewer. "test X for AC-N failed" beats "bundled test failed on assertion 3 of 7" for single-person maintenance. Decisions Pedro made on test granularity reflect this economics — not a universal rule, but a durable project convention.

6. **Dynamic UI epics need a dedicated failure-mode category.** Tailwind CDN JIT, Portuguese locale formatting, ARIA state machines, clipboard permission variance, 1.5s/3s navigation timing — none of these patterns appeared in backend epics 1-4. Epic 5 was the first dynamic-UI epic and surfaced all of them. Epic 6 will surface more (data-driven colour classes, responsive breakpoints, toggle state persistence, skeleton→loaded transitions). Worth naming this category explicitly so retros can track its queue separately from "backend correctness" issues.

## Action Items

### Process Improvements

**P1. Test-granularity convention — per-scenario default + DOM-smoke carve-out.** Owner: Pedro (memory entry). Epic 6 default: per-scenario Playwright tests (one `test()` per scenario ID in the test plan). Carve-out: DOM smoke tests can bundle since they're structural-presence, not behavioural. Reasoning: solo-dev debug economics (failure pinpointing), Playwright worker parallelism (one test = one worker), plan-to-test 1:1 mapping (no mental translation), dev-agent ergonomics (matches test plan verbatim), `/bad-review` audit clarity (coverage-by-scenario-ID). Ship as `feedback_test_granularity.md` memory.

**P2. Shared frontend architecture invariants — extract from story ATDDs to `tests/frontend-architecture-invariants.test.js`.** Owner: Pedro. Move the 3 architecture invariants currently duplicated in Stories 5.1/5.2 ATDDs (no `eval`/`document.write`, no server-side imports, no `innerHTML` user injection) into a shared file that greps all `public/js/*.js`. Story ATDDs shrink back to AC-mapped only (matches original Epic 4 retro rule cleanly, no exception-carving needed). Success: Epic 6 story ATDDs stay AC-scoped; shared file catches any `public/js/*.js` drift. Ship as `feedback_frontend_architecture_invariants.md` memory.

**P3. BAD Step 6 PR-body prompt — count-category labelling tweak.** Owner: Pedro. Epic 5's Playwright scaffolds (`84e2e96`) validated the pattern. Epic 6 prep will repeat it: `report.smoke.spec.js` with `test.skip` templates for 6.1-6.6 + DOM smoke, ships in the same prep PR. ### Technical / Scope

**T1. Tailwind dynamic-class rule — codify as invariant + static scan.** Owner: Pedro. Rule: any class name that appears only in a JS string literal (e.g. `classList.add('bg-red-600')`) must have either (a) a sibling static occurrence in the HTML, or (b) an inline style fallback at the write site. Reason: Tailwind Play CDN JIT purges class names with no static reference. Enforcement: static scan in `tests/frontend-architecture-invariants.test.js` (P2) greps `classList.add\|classList.remove\|className =` tokens and cross-references against HTML static occurrences + inline style fallbacks. Ship as `feedback_tailwind_dynamic_classes.md` memory.

**T2. pt-PT locale assertion helpers.** Owner: Pedro. Add a 10-line helper module at `tests/e2e/test-helpers.js` exposing `expectPortuguesePrice(locator, value)` and `expectPortugueseDate(locator, date)`. Epic 6 stories 6.1/6.2/6.3 all format dates and money; helper standardises the assertion strings (`€799,00`, `14 de Abril de 2026`). Ship in the bundled prep PR.

**T3. CTA_URL deploy-time placeholder guard — land in Story 6.3 as extra AC.** Owner: Pedro (adding to 6.3 spec). Deferred since Story 1.1 code review — four-epic lifetime without a natural landing point. Story 6.3 touches `CTA_URL` const at top of `report.js` anyway; the cheapest retrofit window we'll ever get. Landing plan: add AC to Story 6.3 spec — "If `CTA_URL` equals `'TODO'` or contains the literal `'placeholder'`, log a startup warning (`fastify.log.warn`) on dev-server boot — non-fatal, just visible." Static scan in `tests/frontend-architecture-invariants.test.js` (P2) doubles-checks the placeholder never makes it into a build.
